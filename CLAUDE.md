## Project Overview

This is a **UVM testbench** for a UART module with an APB2 slave register interface. The RTL implements a configurable UART (baud rate, packet width, parity, stop bits) with dual 8-deep FIFOs. The testbench uses the UVM Register Abstraction Layer (RAL) for APB2 register access.

## Simulation Commands

No Makefile exists. Compile and simulate manually with your SystemVerilog simulator. Example using VCS:

```bash
# Compile
vcs -full64 -sverilog -ntb_opts uvm \
    +incdir+./tb +incdir+./rtl \
    ./rtl/*.sv ./rtl/*.v ./tb/*.sv \
    -top tb_top -debug_all -o simv

# Run a specific test
./simv +UVM_TESTNAME=uart_base_test

# Run with verbosity
./simv +UVM_TESTNAME=uart_parity_test +UVM_VERBOSITY=UVM_HIGH
```

For Questa/ModelSim, replace `vcs` with `vlog`/`vsim`.

### Data Flow

**TX path**: Sequence item в†’ TX driver writes FIFO в†’ UART serializes в†’ TX monitor captures serial output в†’ Scoreboard compares against TX input monitor (FIFO write snooping).

**RX path**: Sequence item в†’ RX driver transmits serial frames в†’ UART deserializes into RX FIFO в†’ RX monitor reads FIFO в†’ Scoreboard compares against RX input monitor (serial input snooping).

**APB/Config path**: APB agent writes CONFIG0/CONFIG1 registers via RAL в†’ drives UART control signals.

### RAL Register Map (`uart_regs_uvm_pkg.sv` вЂ” auto-generated)

| Reg | Addr | Key Fields |
|---|---|---|
| CONFIG0 | 0x0 | `rx_en`, `tx_en` [7:6], `uart_sm` [5:1], `parity_en` [0] |
| CONFIG1 | 0x1 | `parity_odd_even` [4], `packet_width` [3:2], `stop_bit_count` [1:0] |
| STATUS0 | 0x2 (RO) | `rx_parity_error`, `rx_frame_error`, FIFO write/read errors |
| STATUS1 | 0x3 (RO) | `tx_fifo_full`, `rx_fifo_full`, `rx_fifo_empty` |

### Key Configuration (`uart_config.sv`)

- `uart_sm` [0вЂ“19]: baud rate selector; `cycles_per_bit = get_count_top(uart_sm) Г— 10`
- Default constraint: `uart_sm == 19` (fastest, for simulation speed)
- `packet_width`: 2'b00вЂ“2'b11 в†’ 5вЂ“8 data bits
- `num_transactions`: number of items per sequence (default 10)

### Testbench Interfaces

Four virtual interfaces set in `uvm_config_db` during `initial` block in `testbench.sv`:
- `uart_if` вЂ” serial TX/RX lines
- `uart_fifo_if` вЂ” FIFO write/read signals
- `apb_if` вЂ” APB2 protocol bus
- `intr_if` вЂ” interrupt signal

Clock: 100 MHz (PCLK). Reset: active-low PRESETn, released at 100 ns. Simulation timeout: 5 ms.

### Scoreboard Completion

`uart_scoreboard` signals `tx_done`/`rx_done` events when match count reaches expected transaction count. `uart_base_test::run_phase` waits on both before dropping objections.

### Adding a New Test

1. Extend `uart_base_test` in `tb/uart_test_lib.sv`
2. Override constraints on `cfg` (uart_config) in `build_phase` if needed
3. Override `run_phase` to call specific sequences or add APB sequences
4. Register with `` `uvm_component_utils ``
